# Astra UI 验证 Checklist

在交付任何页面、组件或原型前，请务必对照以下清单进行合规性验证。

## Critical - 极高优先级 (必须通过)

### 导航与结构
- [ ] **一级导航**: 宽度必须为 110px, 背景必须为 `bg-slate-900`。
- [ ] **向排列**: 一级导航项图标+文字必须**横向等比排列** (禁止竖排或折行)。
- [ ] **二级导航**: 宽度必须为 130px, 背景必须为纯白 `bg-white`。
- [ ] **面包屑**: 页面顶部必须有固定背景为 `bg-white` 的面包屑 / Header 栏。

### 布局与间距
- [ ] **Canvas 背景**: 页面底色必须是 `bg-slate-50`。
- [ ] **内容载体**: 所有业务内容必须承载在 `bg-white` 且 `rounded-xl` 的卡片中，**严禁**内容直接悬浮在底色上。
- [ ] **安全距离**: 内容区域边距必须统一为 `p-5` (20px)。

### 颜色与表面
- [ ] **无描边规则**: 卡片容器**严禁出现描边** (`border`)。
- [ ] **无阴影规则**: 卡片容器**严禁出现外阴影** (`shadow`)，仅 Header 或 弹出层允许使用极轻量阴影。

## Major - 高优先级 (设计合规)

### 表格与操作
- [ ] **操作可见性**: 表格操作列**不可默认隐藏**。
- [ ] **操作平铺规则**: 操作数量 ≤ 3 个时，必须全部外露平铺展示。
- [ ] **收纳规则**: 操作数量 > 3 个时，超出的项必须收纳至 “更多” 下拉菜单，避免堆叠。
- [ ] **视觉一致性**: 操作项文字**严禁混搭多种颜色** (如红+灰)，必须统一使用品牌蓝 (`text-blue-600`) 且字重为 400 (`font-normal`)。

### 按钮与交互
- [ ] **按钮比例**: 核心 CTA (如新建) 放在卡片左侧首位。
- [ ] **圆角体系**: 使用 `rounded-md`, 全面**禁用** `rounded-full` 圆柱形按钮。
- [ ] **图标一致性**: 必须使用 `lucide-react` 图标，`strokeWidth` 统一为 2.5。
- [ ] **导航分隔**: 导航区域之间不使用 `border`、`border-r` 等分隔线。

## 审计流程
1. **开发者自检**: 使用此 Checklist 完成 100% 匹配。
2. **Page Agent 校验**: 运行自动化脚本验证 DOM 结构与样式类名。
3. **视觉走查**: 确保没有多余的装饰性元素 (渐变、彩色边框)。
